# Repository Guidelines

## 작업 환경
- 운영체제: Microsoft Windows 11 Pro 10.0.26100
- Shell: PowerShell 5.1.26100.8655
- 프로젝트 루트: `D:\workspace\codex_edu\codex_practice0`
- 주요 런타임: `node 24.16.0`, `git 2.54.0.windows.1`, `python UNKNOWN`, `npm 11.17.0`

## 기본 원칙
이 저장소는 단일 정적 페이지 프로젝트다. Codex는 이 프로젝트 범위 안에서만 작업하고, 이후 설명, 계획, 보고는 기본적으로 한국어로 한다.

1. 프로젝트 내부 파일을 직접 읽고 판단한다.
2. 추정으로 구현하지 않는다.
3. 요구사항이 모호하면 먼저 짧게 확인하거나, 확인이 어려우면 안전한 최소 범위로 작업한다.
4. 필요한 경우 먼저 짧은 계획을 세운 뒤 작업한다.
5. 코드 수정 후 관련 테스트 또는 최소 검증을 수행한다.
6. 테스트를 실행할 수 없으면 그 이유를 명확히 기록한다.
7. 변경 내용과 검증 결과를 마지막에 짧게 요약한다.
8. `.env`, secret, credential, API key, 인증 파일은 출력하거나 수정하지 않는다.
9. 프로젝트 스코프 밖 파일은 읽거나 수정하지 않는다.
10. 삭제, 강제 덮어쓰기, 원격 push 같은 파괴적 작업은 수행하지 않는다.
11. 사용자가 제공한 변수명, 함수명, 파일명은 임의로 바꾸지 않는다.
12. 불필요한 대규모 리팩토링은 하지 않는다.
13. 요청 범위 밖의 기능은 임의로 추가하지 않는다.

## 프로젝트 구조
현재 핵심 파일은 루트의 `index.html` 하나다. 별도의 소스, 테스트, 빌드 디렉터리는 없다. 지원 파일이 필요하면 `assets/`, `images/`, `scripts/`처럼 단순한 폴더를 사용한다.

## 작업 방식
개발 도구가 없으므로 `index.html`을 브라우저로 직접 확인한다. 변경 후에는 `git diff`로 범위를 점검하고, 화면이 있는 변경은 데스크톱과 모바일 폭에서 직접 확인한다.

## 커밋과 PR
커밋 메시지는 현재처럼 짧고 소문자 중심으로 유지한다. PR에는 변경 요약, 시각적 변경의 경우 스크린샷, 수행한 검증 결과를 포함한다.
